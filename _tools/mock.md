# Tool: `mock <type>`

Timed, realistic mock interview round + debrief. Types: `coding`, `ml-theory`, `system-design`, `recsys`, `behavioral`, `mixed`.

## Behavior — run it like the real thing
1. State the format up front: duration, what's expected. Then **start the clock** (note start time; Soham reports when done, or you time the back-and-forth).
   - coding: 1 medium in ~35 min, talk-aloud expected.
   - ml-theory: 4–6 rapid-fire + deep follow-ups, ~30 min.
   - system-design: 1 open prompt, ~45 min, you play interviewer.
   - recsys: a personalization design prompt, ~45 min.
   - behavioral: 4–5 STAR questions at Senior scope, ~30 min.
   - mixed: 1 coding + 1 short design.
2. **Stay in character as the interviewer.** Push back. Ask "why", "what about at 50M users", "what would you do differently". Don't help unless they're truly stuck (and note that they needed a hint).
3. Do **not** grade mid-round. Take notes silently.
4. At the end → full **debrief**:
   - Score per pillar dimension (rubric).
   - Hire signal: Strong No / No / Lean / Yes / Strong Yes — and *why*, at the Senior bar.
   - Top 2 things that would have changed the verdict.
   - What to drill next.
5. Save transcript → `07_mock_interviews/transcripts/YYYY-MM-DD_<type>.md`; debrief → `07_mock_interviews/debriefs/YYYY-MM-DD_<type>.md`.
6. Update tracker for every topic touched.

> The hire-signal call must be honest. A "Lean No" now is worth more than a fake "Yes."
